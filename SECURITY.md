# Security Policy

If you discover a security issue, please do not open a public GitHub issue with exploit details.

Instead:

1. Prepare a short report with reproduction steps and impact.
2. Share it privately with the project maintainers through a non-public channel.
3. Allow time for a fix before public disclosure.

## Scope

Please report issues involving:

- Authentication and authorization
- Account takeover
- Sensitive data exposure
- Payment and subscription flows
- Push notifications and messaging
- Infrastructure or deployment misconfiguration
